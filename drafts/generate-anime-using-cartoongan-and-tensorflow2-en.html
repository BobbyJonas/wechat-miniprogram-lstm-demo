Generate Anime using CartoonGAN and TensorFlow 2.0

  • Sun, 05 May 2019
  • 1 views

Which image will you choose to transform if you can turn any image into anime using AI?

In this post, I will briefly introduce a CartoonGAN implemented in TensorFlow 2.0 Alpha by my friend mnicnc404 and me.

We will also demo a TensorFlow.js app which allow you to generate anime on the browser directly.

CartoonGAN (original paper) was published in 2018 CVPR and is a Generative Adversarial Network (GAN) which attempt to transform real world images into cartoon-style images.

Left-top is real world image, while all other 3 images are generated by CartoonGAN using different anime styles

The ideas demonstrated in the paper are very interesting and the authors also show good results using two of my favorite Japanese animators' styles: Shinkai Makoto and Miyazaki Hayao.

Therefore, I decided to build a TensorFlow.js app which allow everyone to try CartoonGAN with ease.

Choose your favorite anime style and upload one image, just that simple.


Under the hood, TensorFlow.js run in your browser and it will:

  • download the pretrained models
  • cartoonize uploaded images

To get the best experience, it is highly recommended that you:

  • try the app with desktops/laptops rather than mobile phones
  • try the app when network speed is fast（to minimalize loading time）

You got the idea. All you have to do is open the notebook and generate your anime in a breeze.

Marvel movies, cute cats, scenery and popular idols, the sky is the limit. When trained, we can use CartoonGAN to combine everything with anime.

Using python script in our github repo, you can generate these results with one command:

python cartoonize.py \
    --styles shinkai hayao hosoda

You can imagine apps with similar features will become much more powerful in the near future. Everyone will be able to generate their own anime, while animators can test their new ideas and draw drafts much more faster than now.

Generative Models like CartoonGAN shown here can broaden our horizons and show us much more possibilities.

Train your own CartoonGAN¶

Yes, you may noticed that we put a lot of focus on the actual application rather than the algorithm details for CartoonGAN. And it’s just because:

  • I believe most of the readers only interested in how to use CartoonGAN rather than train a Model themselves
  • There are already lots of learning resources on the web

For those who are interested in implementation details of our CartoonGAN, we suggest you taking a look at the github repo. If you have your own datasets, you can even train your own CartoonGAN simply in one command:

Our Python script provide detailed messages which allow you to understand what is happening

The repo can also be used to learn latest version of TensorFlow since everything is implemented in Tensorflow 2.0 alpha:

  • Use tf.keras to implement custom layers and GAN model
  • Use tf.data to load and process large amount of images
  • Write custom train steps and make computation faster using tf.function
  • Use TensorBoard to monitor model performance in real-time manner

As some of you already knew, it is not trivial to train a GAN properly. You will need to keep monitoring model performance to decide how to adjust your hyperparameters.

In our training script, TensorBoard is integrated so you can monitor model performance in real-time manner:

Result of one of our experiments on TensorBoard

In addition to metrics and loss functions, it is a good practice to keep an eye on the images that the GAN is generating.

Using our script, you can check generated images on TensorBoard as well:


Although we only record cartoonGAN-specific metrics and images here, you can apply same technique to monitor any other models.

Finally, in order to get a sense of how CartoonGAN is doing during training phase, we can save some images and use them as validation set.

The idea here is to "test" CartoonGAN periodically so that we can see how the model perform over time:

CartoonGAN become better and better transforming validation set into anime

You can see that CartoonGAN learn to transform real world images into cartoon-like images with clear edges and smooth color shading.

Training a GAN is far more harder than training a simple classifier. But in our github repo, we make an effort to make our code clear and easy to understand, hoping it will help more people to get started.

Indeed, you can get a better understanding of our code if you know some basics. In the next section, I will list up some good learning resources for reference.

Recommended learning resources¶

Recently, deep learning-based generative models are very popular and GAN is one of the most exciting research areas. The CartoonGAN we saw in this post is just a simple application in the big GAN zoo.

In a typical GAN setting, there are two independent neural networks, called generator and discriminator, competing with each other （圖片來源）

For those who want to learn more about GAN, I recommend following resources:

  • MIT 6.S191 Deep Generative Models
  • GAN Lab let you train GANs on browser
  • Andrej Karpathy's online GAN demo
  • TensorFlow offical tutorial teach you how to generate MNIST using DCGAN
  • Synced: Reproducing Japanese Anime Styles With CartoonGAN AI
MIT 6.S191 Deep Generative Models is a good place to start learning GAN concepts （圖片來源）

The researchs on generative models and GAN are evolving rapidly, but I believe you will be able to implement your own GAN after studying these learning resources.

Final words¶

AI technology and applications like CartoonGAN should be more accessable for everyone, not only researchers or machine learning practitioners.

This is my first GAN-related project using TensorFlow 2.0 and TensorFlow.js and I learnt a lot from it. Many thanks to the authors of CartoonGAN, TensorFlow team and my friend mnicnc404.

Hope this project and blog post can encourge people to explore more about GAN and generative models, create more interesting AI applications in the future.

Oh! Also don't forget to share what anime you generate with me! You can tag me on Twitter or Facebook.

Happy cartoonizing! :D
