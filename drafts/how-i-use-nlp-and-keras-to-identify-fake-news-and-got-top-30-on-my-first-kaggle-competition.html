我如何利用自然語言處理以及 Keras 在第一次 Kaggle 競賽達到前 30 %

  • 2018-12-17 (Mon)
  • 1 views

自從 11 月從比利時 EMNLP 回來後，最近工作之餘都在學習自然語言處理（Natural Language Processing, 後簡稱為 NLP）。依據維基百科的定義：

自然語言處理（NLP）是計算機科學以及人工智慧的子領域，專注在如何讓計算機處理並分析大量（人類的）自然語言數據。NLP 常見的挑戰有語音辨識、自然語言理解、機器翻譯以及自然語言的生成。

在這篇「短篇小說」裡頭，我將描述自己怎麼陰錯陽差地參加一個辨別假新聞的 Kaggle 競賽，並如何利用最近學到的 NLP 知識以及深度學習框架 Keras 來教會神經網路（Neural Network）自動辨別手中的假新聞。

後面你會看到，這些模型表現得還不錯！


在這篇文章裡頭，我會盡量用簡單的詞彙來解釋故事中出現的深度學習及 NLP 概念，同時也會在文末附上不少實用的學習資源以及文章供你進一步探索。

本文希望能讓對 AI、深度學習或是 NLP 有興趣，但還不知道怎麼在這篇汪洋大海航行的你，在閱讀本故事以後找出一些明確的方向，展開自己的 NLP 之旅。

意料之外的 Kaggle¶

本文提到的 Kaggle 競賽是 WSDM - Fake News Classification。

此競賽的目的在於想辦法自動找出假新聞以節省人力成本。資料集則是由中國的手機新聞應用：今日頭條的母公司字節跳動所提出的。（知名的抖音也是由該公司的產品）

本文的 Kaggle 競賽 （圖片來源）

而因為我所任職的 SmartNews 的主打產品也是手機新聞應用（針對日本與美國用戶），像是這種哪個企業又辦了 Kaggle 競賽、又開發什麼新功能等等的消息都會在公司內部流動。

話雖如此，在我從同事得知這個為期一個月的競賽時，事實上離截止時間只剩一個禮拜了！（傻眼）

今年 10 月底參加的 EMNLP （圖片來源）

名符其實的「志在參加」。

假新聞分類問題¶

既然決定要參加了，當然得看看資料集長的什麼樣子。訓練資料集（Training Dataset）約有 32 萬筆數據，而測試資料集（Test Dataset）則約為 8 萬筆。而訓練資料集實際的內容如下：


要了解此資料集，讓我們先專注在第一行（Row），大蒜與地溝油新聞的每一個欄位。

第一欄 title1_zh 代表的是「已知假新聞」 A 的中文標題：

用大蒜鉴别地沟油的方法,怎么鉴别地沟油

而第二欄 title2_zh 則是一筆新的新聞 B 的中文標題，我們還不知道它的真偽：

翻炒大蒜可鉴别地沟油

要判斷第二欄中的新聞標題是否為真，我們可以把它跟已知的第一篇假新聞做比較，分為 3 個類別：

  • unrelated：B 跟 A 沒有關係
  • agreed：B 同意 A 的敘述
  • disagreed：B 不同意 A 的敘述

如果 B 同意 A 的敘述的話，我們可以將 B 也視為一個假新聞；而如果 B 不同意假新聞 A 的敘述的話，我們可以比較放心地將新聞釋出給一般大眾查看；如果 B 與 A 無關的話，可以再進一步處理。

如果 B 新聞「同意」假新聞 A 的話，我們大可將 B 新聞也視為假新聞，最後將其屏除

接著看到下圖第一行最右邊的 label 欄位為 agreed，代表 B 同意 A 的敘述，則我們可以判定 B 也是假新聞。這就是一個簡單的「假新聞分類問題」：給定一個成對的新聞標題 A & B，在已知 A 為假新聞的情況下，預測 B 跟 A 之間的關係。其關係可以分為 3 個類別：

  • unrelated
  • agreed
  • disagreed

順帶一提，上圖同時包含了 3 個類別的例子供你參考。

第 3、 4 欄位則為新聞標題的英文翻譯。而因為該翻譯為機器翻譯，不一定能 100% 正確反映本來中文新聞想表達的意思，因此接下來的文章會忽視這兩個欄位，只使用簡體中文的新聞標題來訓練模型。

用直覺找出第一條底線¶

現在目標很明確了，給定一個假新聞標題 A，我們就是要利用機器學習（Machine Learning, ML）來準確判斷新聞 B 跟新聞 A 的關係，減少人工檢查的成本並同時最大化分類的精準度。

但在開始使用任何 ML 方法之前，為了衡量我們的自動化模型能提供多少潛在價值，讓我們先找出一個簡單方法作為底線（Baseline）。

這張圖顯示了訓練資料集（Training Dataset）裏頭，各個分類的數據所佔的比例。常見的 Unbalance Dataset。

我們可以看到接近 70 % 的新聞 pairs 都是不相關的。（新聞 pairs 指的是資料集裡，每一行的新聞標題 A 以及 標題 B 所組成的 pairs）

假設測試資料集（Test Dataset）的數據分佈跟訓練資料集相差不遠，且衡量一個分類模型的指標是準確度（Accuracy），這時候如果要你用一個最簡單的模型來分類所有新聞 pairs 並同時最大化準確度，你會怎麼做？


對沒錯，就是全部猜 unrelated 就對了！

事實上，此 Kaggle 的 Evaluation 是使用 Weighted Categorization Accuracy，來調降猜對 unrelated 的分數。畢竟（1）能正確判斷出兩個新聞是 unrelated 跟（2）能判斷出新聞 B disagreed 假新聞 A 的價值是不一樣的。（後者價值比較高）

但使用多數票決（Majority Votes）的簡單方法至少能得到 0.666 的成績（滿分為 1）：


不過你去該 Kaggle 排行榜的話卻會發現不少人低於這個標準：


第一次參加 Kaggle 的人可能會覺得這現象很奇怪。

但這是由於 Kaggle 競賽 1 天只能提交 2 次結果，因此通常不會有人浪費提交次數來上傳「多數票決」的結果（儘管分數會上升，大家還是會想把僅僅 2 次的上傳機會用來測試自己的 ML 模型的準確度）；另外也是因為不少人是上傳 1、2 次就放棄比賽了。

但如果你的 ML 或深度學習模型怎樣都無法超過一個簡單 baseline 的話，或許最後上傳該 baseline 的結果也不失為提升排名的最後手段（笑）

找出 Baseline，可以讓我們判斷手上訓練出來的機器學習模型有多少潛在價值、值不值得再繼續花費自己的研究時間與電腦計算能力。

現在我們知道，要保證做出來的模型有點價值，最少要超過 baseline 才可以。以本文來說，就是多數票決法則得到的 0.666 準確度。

（ baseline 的定義依照研究目的以及比較方法而有所不同）

資料前處理：讓機器理解文字¶

要讓電腦或是任何 NLP 模型理解一篇新聞標題在說什麼，我們不能將自己已經非常習慣的語言文字直接扔給電腦，而是要轉換成它熟悉的形式：數字。


因此這章節就是介紹一系列的數據轉換步驟，來將人類熟悉的語言如：

用大蒜鉴别地沟油的方法,怎么鉴别地沟油

轉換成人腦不易理解，但很「機器友善」的數字序列：

[217, 1268, 32, 1178, 25, 489, 116, 4]

如果你對此步驟已經非常熟悉，可以假設我們已經做完資料前處理，並直接跳到下一章節的我的 NLP 學習 Timeline。

這章節的數據轉換步驟包含：

  • 文本分詞（Text Segmentation）
  • 建立字典並將文本轉成數字序列
  • 序列的 Zero Padding
  • 將正解做 One-hot Encoding

如果你現在不知道上述所有詞彙的意思，別擔心！

文本分詞¶

文本分詞（Text Segmentation）是一個將一連串文字切割成多個有意義單位的步驟。這單位可以是

  • 一個中文漢字 / 英文字母（Character）
  • 一個中文詞彙 / 英文單字（Word）
  • 一個中文句子 / 英文句子（Sentense）

依照不同的 NLP Task 會有不同需求，但很常使用的切法是以單字（Word）為單位，也就是 Word Segmentation。

以英文來說，想做 Word Segmentation 簡單到不行。通常只要依照空白分割，就能得到一個有意義的詞彙列表了（在這邊讓我們無視標點符號）：

text = 'I am Meng Lee, a data scientist based in Tokyo.'
words = text.split(' ')
words

['I', 'am', 'Meng', 'Lee,', 'a', 'data', 'scientist', 'based', 'in', 'Tokyo.']

但中文與日文都無法這樣做。這時候我們將藉助如 Jieba 等中文斷詞工具幫我們為一連串的文字做有意義的切割：

import jieba.posseg as pseg

text = '我是李孟，在東京工作的數據科學家'
words = pseg.cut(text)
[word for word in words]

[pair('我', 'r'),
 pair('是', 'v'),
 pair('李孟', 'nr'),
 pair('，', 'x'),
 pair('在', 'p'),
 pair('東京', 'ns'),
 pair('工作', 'vn'),
 pair('的', 'uj'),
 pair('數據', 'n'),
 pair('科學家', 'n')]

如上所示，Jieba 將我們的中文文本切成有意義的詞彙列表，並為每個詞彙附上對應的詞性（Flag）。

假設我們不需要標點符號，則只要將 flag == x 的詞彙去除即可。

我們可以寫一個很簡單的 Jieba 斷詞函式，此函式能將輸入的文本 text 斷詞，並回傳除了標點符號以外的詞彙列表：

def jieba_tokenizer(text):
    words = pseg.cut(text)
    return ' '.join([
        word for word, flag in words if flag != 'x'])

我們可以利用 Pandas 的 apply 函式，將 jieba_tokenizer 套用到所有新聞標題 A 以及 B 之上，做文本分詞：

train['title1_tokenized'] = \
    train.loc[:, 'title1_zh'] \
         .apply(jieba_tokenizer)
train['title2_tokenized'] = \
    train.loc[:, 'title2_zh'] \
         .apply(jieba_tokenizer)

建立字典並將文本轉成數字序列¶

當我們將完整的新聞標題切成一個個有意義的詞彙（Token）以後，下一步就是將這些詞彙依照出現順序轉換成一個數字序列，方便電腦處理。

這些數字是所謂的索引（Index），分別對應到特定的詞彙。


為了方便你理解這小節的概念，現在想像個極端的例子。假設我們就只有一個新聞標題：「狐狸被陌生人拍照」。

這時候要怎麼將這個句子轉成一個數字的序列呢？跟上一小節相同，我們首先會對此標題做斷詞，將句子分成多個有意義的詞彙：

text = '狐狸被陌生人拍照'
words = pseg.cut(text)
words = [w for w, f in words]
words

['狐狸', '被', '陌生人', '拍照']

有了詞彙的列表以後，我們可以建立一個字典 word_index。

該 dict 裏頭有 4 個詞彙當作鍵值（Key），值（Value）則分別對應到不重複的數字：

word_index = {
    word: idx  
    for idx, word in enumerate(words)
}

words = pseg.cut(text)

在這個簡單的狐狸例子裡頭，word_index 就是我們的字典；我們利用該字典，將 1 句話轉成包含多個數字的序列，而每個數字實際上代表著一個 Token。

同理，現在我們可以分 3 個步驟將手上的新聞標題全部轉為數字序列：

  1. 將被斷詞的新聞標題 A 以及新聞標題 B 全部倒在一起
  2. 建立一個字典，為這些新聞標題裡頭出現的每個詞彙指定一個索引數字
  3. 利用建好的字典，將每個新聞標題轉換成數字序列

這種文字前處理的使用頻率因為實在太過頻繁，Keras 有專門的文字前處理模組來提升我們的效率：

import keras
MAX_NUM_WORDS = 10000
tokenizer = keras \
    .preprocessing \
    .text \
    .Tokenizer(num_words=MAX_NUM_WORDS)

Tokenizer 顧名思義，即是將一段文字轉換成一系列的詞彙（Tokens），並為其建立字典。這邊的 num_words=10000 代表我們限制字典的大小只包含 10,000 個詞彙，一旦字典達到這個大小以後，剩餘的新詞彙都會被視為 Unknown，以避免字典過於龐大。

如同上述的步驟 1，我們得將所有包含 A 及 B 的新聞標題聚集起來，為它們建立字典：

corpus_x1 = train.title1_tokenized
corpus_x2 = train.title2_tokenized
corpus = pd.concat([
    corpus_x1, corpus_x2])

序列的 Zero Padding¶

雖然我們已經將每個新聞標題的文本轉為一行行的數字序列，你會發現每篇標題的序列長度並不相同：

最長的序列甚至達到 61 個詞彙：

max_seq_len = max([
    len(seq) for seq in x1_train])
max_seq_len

61

而為了方便 NLP 模型處理，一般我們會設定一個 MAX_SEQUENCE_LENGTH，讓所有序列的長度一致。

長度超過此數字的序列會被刪掉多餘部分；而原來的長度不足的序列我們則會在詞彙前面補零。Keras 一樣有個方便函式 pad_sequences 幫我們完成這件事情：

MAX_SEQUENCE_LENGTH = 20
x1_train = keras \
    .preprocessing \
    .sequence \
    .pad_sequences(x1_train, 
                   maxlen=MAX_SEQUENCE_LENGTH)

x2_train = keras \
    .preprocessing \
    .sequence \
    .pad_sequences(x2_train, 
                   maxlen=MAX_SEQUENCE_LENGTH)

一般來說 MAX_SEQUENCE_LENGTH 可以設定成最長序列的長度（此例中的 61）。但這邊為了讓模型可以只看前 20 個詞彙就做出判斷以節省訓練時間，我們先暫時使用 20 這個數字。

讓我們看看經過 Zero Padding 的第一篇假新聞標題 A 變成什麼樣子：

         25,  489, 2877,  116, 5559,    4, 1850,    2,   13], dtype=int32)

你可以清楚看到，因為該新聞標題原本的序列長度並沒有達到剛剛設定的 MAX_SEQUENCE_LENGTH，因此在總長度為 20 的序列中，前面 6 個值被 Keras 補上 0 以說明前 6 個詞彙並不存在。

  • [0, 1, 0] 代表 label 為 agreed
  • [0, 0, 1] 代表 label 為 disagreed

切割訓練集 & 驗證集¶

這部分很簡單，我們只需決定要將整個資料集的多少比例切出來，當作驗證集（Validation Set）。此例中我們用 10 %。

一般來說，我們只在訓練集（Training Set）訓練模型，並用模型沒看過的驗證集來模擬該模型在真實世界的表現。

等到模型在驗證集也表現得夠好後，便在測試集（Test Set）進行最後一次的預測並上傳到 Kaggle。

要切訓練集 / 驗證集，scikit-learn 的 train_test_split 函式是一個不錯的選擇：

from sklearn.model_selection \
    import train_test_split

VALIDATION_RATIO = 0.1
RANDOM_STATE = 9527

x1_train, x1_val, \
x2_train, x2_val, \
y_train, y_val = \
    train_test_split(
        x1_train, x2_train, y_train, 
        test_size=VALIDATION_RATIO, 
        random_state=RANDOM_STATE
)

在這邊，我們分別將新聞標題 A x1_train、新聞標題 x2_train 以及標籤 y_train 都分成兩個部分：訓練部分 & 驗證部分。

我的 NLP 學習 Timeline¶


  • Encoder-decoder : sequence generation
  • Attention: 讓 Decoder 決定要看 encoder 哪部分來做翻譯
  • Transfomer
    • 讓 Encoder 利用 SELF-attention 來 better encode 一個字（The animal didn't cross the street because it was too tired）
    • multi-attention head. 相較於 RNN, transformer 在同個時間點放入整個 seq, 每個 token 處理為獨立，因此能平行化加快運算，解決一個 RNN 痛點。但因為這樣就沒有隱含類似 RNN 不同 timestep 的 seq order，因此除了 token 本身的 embed 以外會加上 positional embed

  • BERT

    • 所以你現在可以想像 BERT 名稱裡頭的數字代表什麼：L 為層數，A 代表 #attention_head ，一億參數

  • summary: Simple RNN —> LSTM —> encoder-decoder —> attention-based model —> transformer —> BERT
